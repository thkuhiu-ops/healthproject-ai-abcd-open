# Historical ABCD v0.2 Final Action Mapping

This file preserves historical ABCD v0.2 action-mapping terminology.
The current default deployment candidate should be described as **ABD integrated / ABD_SAFE**, where final trustworthy-output behavior is generated by the firmware Fusion arbitration layer.

## Current Default Output Semantics

The default ABD integrated / ABD_SAFE candidate uses engineering state classes such as:

| ABD_SAFE State | Meaning |
| --- | --- |
| `FINAL_OK` | Signals and rule states are sufficient for an engineering-level accepted measurement state. |
| `SIGNAL_BAD_OR_CONTACT_BAD` | Signal quality, contact, lead, or related rule state indicates retest/contact improvement is needed. |
| `UNCERTAIN_OR_MOTION` | Motion, uncertainty, or conflicting evidence indicates the measurement should not be trusted directly. |

The firmware Fusion layer converts model-assisted evidence and hard rules into `trust_score`, `trust_level`, `final_state`, `hr_source`, and UI status.
